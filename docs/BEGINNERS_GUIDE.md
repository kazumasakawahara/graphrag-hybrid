## 1. このシステムは何をしてくれるのか

### ひとことで言うと

GraphRAG Hybrid は、**あなたの手元にあるドキュメント（PDF、報告書、マニュアルなど）を「意味で検索できる」ようにするシステム**です。

普通の検索は「完全に同じ単語」が含まれている文書しか見つけられません。たとえば「成年後見」と検索すると、「成年後見」という文字列が書かれた文書だけがヒットします。「判断能力が不十分な方の権利を守る制度」と書かれた文書は見つかりません。

GraphRAG Hybrid は違います。**文章の「意味」を理解して検索します。** さらに、ドキュメント同士の「関係性」や、文書に登場する「人物・組織・概念」のつながりも把握します。

> 図書館の司書のように、あなたの質問の意味を理解し、関連する資料も含めて最適な情報を見つけ出します。

### 3つの検索を組み合わせるハイブリッド方式

このシステムは、3つの異なる検索方法を組み合わせて、より正確な結果を返します。

| 検索方式 | 割合 | 何をしているか |
|---------|------|--------------|
| セマンティック検索 | 60% | 文章の「意味」の近さで検索します |
| グラフ検索 | 20% | ドキュメント間の「関係性」をたどって検索します |
| エンティティ検索 | 20% | 文書に登場する「人物・組織・概念」から検索します |

### こんな場面で役立ちます

#### 福祉の現場

条文番号や正式名称を覚えていなくても、自然な言葉で検索できます。

- 「親が亡くなった後の障害者の生活支援について」と入力するだけで、関連する法令・制度情報・事例をまとめて見つけられます
- 「認知症の方の財産管理」と検索すると、成年後見制度、日常生活自立支援事業、家族信託など、関連する制度の情報が横断的にヒットします
- 利用者のケースに似た過去の支援記録を探すことができます

#### 法律事務所

- 判例や法令を「意味」で検索し、関連する裁判例も自動で提示されます
- 「賃貸人が修繕義務を怠った場合の賃借人の救済手段」のような自然文で検索できます
- ある法律概念に関連するすべての文書を、エンティティのつながりからたどれます

#### 研究者

- 論文やフィールドノートを一元管理し、概念間の関係性を把握できます
- 「ソーシャルキャピタルと地域福祉の関係」のようなテーマで横断的に検索できます
- 新しい資料を追加すると、既存の資料との関連が自動的に見つかります

#### 行政

- 膨大な通知・通達・ガイドラインから、必要な情報を瞬時に発見できます
- 「災害時の要配慮者の避難支援」のような実務的な質問で検索できます
- 部署をまたいだ関連文書を一括で見つけられます

## 2. 必要なもの

セットアップに必要なものを順番に説明します。すべて無料で入手できます。

### パソコン

Mac、Windows、Linux のいずれでも使えます。特別に高性能なパソコンは必要ありません。一般的な業務用パソコンで動作します。

### Docker Desktop（必須）

**Docker Desktop は、データベースという「情報の倉庫」をあなたのパソコンの中に作るためのツールです。**

このシステムでは、2つのデータベース（Neo4j と Qdrant）を使います。Docker Desktop を使うと、これらのデータベースをボタンひとつで起動できます。

- ダウンロード: https://www.docker.com/products/docker-desktop/
- Mac の方: ダウンロードした `.dmg` ファイルを開き、Docker のアイコンをアプリケーションフォルダにドラッグするだけです
- Windows の方: ダウンロードしたインストーラーを実行し、画面の指示に従ってください

インストール後、Docker Desktop を起動してください。画面の左下に「Engine running」と緑色で表示されれば準備完了です。

### uv（必須）

**uv は、このシステムに必要な部品を自動的にインストールしてくれるツールです。**

Python というプログラミング言語の環境を管理するためのものですが、プログラミングの知識は必要ありません。

#### Mac / Linux の方

「ターミナル」アプリを開いて、以下をコピー＆ペーストしてください。

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

> **ターミナルの開き方（Mac）**: Finder で「アプリケーション」>「ユーティリティ」>「ターミナル」を開きます。または、Spotlight 検索（Command + Space）で「ターミナル」と入力しても見つかります。

#### Windows の方

「PowerShell」を開いて、以下をコピー＆ペーストしてください。

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

> **PowerShell の開き方**: スタートメニューで「PowerShell」と検索してください。

### Git（必須）

**Git は、システムのファイル一式をダウンロードするためのツールです。**

- Mac の方: ターミナルで `git --version` と入力してください。初回はインストールを促すダイアログが出ますので、指示に従ってインストールしてください。
- Windows の方: https://git-scm.com/download/win からダウンロードしてインストールしてください。

### Claude Desktop（任意・推奨）

**Claude Desktop は、AI アシスタント「Claude」を通じてこのシステムを操作できるアプリです。**

登録したドキュメントに対して、自然な日本語で質問するだけで、Claude が適切な検索を行い、回答を生成してくれます。プログラミングの知識がなくても、会話するだけでシステムのすべての機能を使えます。

- ダウンロード: https://claude.ai/download

### Gemini API キー（任意）

**ドキュメントから人物名・組織名・概念などを自動的に抽出する機能を使いたい場合に必要です。**

この機能を使わなくても、セマンティック検索とグラフ検索は正常に動作します。

- 取得方法: https://aistudio.google.com/ にアクセスし、Google アカウントでログインして API キーを作成してください（無料で取得できます）

## 3. セットアップ手順

以下の手順を上から順番に進めてください。各ステップで実行するコマンドは、ターミナル（Mac/Linux）または PowerShell（Windows）にそのままコピー＆ペーストできます。

### ステップ 1: システムのダウンロード

```bash
git clone https://github.com/kazumasakawahara/graphrag-hybrid.git
cd graphrag-hybrid
```

このコマンドは、GraphRAG Hybrid のファイル一式をあなたのパソコンにダウンロードし、そのフォルダに移動します。ダウンロード先は、ターミナルを開いたときのフォルダ（通常はホームフォルダ）の中に `graphrag-hybrid` という名前で作成されます。

### ステップ 2: 設定ファイルの準備

```bash
cp .env.example .env
```

設定のひな形ファイルをコピーします。通常はこのままの設定で動作しますので、中身を変更する必要はありません。

Gemini API キーをお持ちの方は、コピーした `.env` ファイルをテキストエディタで開き、最後の行の `# GEMINI_API_KEY=your_api_key_here` を以下のように書き換えてください。

```
GEMINI_API_KEY=ここにあなたのAPIキーを貼り付け
```

行の先頭の `#` を消すのを忘れないでください。`#` が付いたままだと設定が読み込まれません。

### ステップ 3: データベースの起動

まず、Docker Desktop が起動していることを確認してください（画面の左下に「Engine running」と表示されていれば OK です）。

```bash
docker compose up -d
```

このコマンドで、2つのデータベースが起動します。

- **Neo4j**: ドキュメント同士の関係性を管理するデータベース
- **Qdrant**: テキストの「意味」を検索するためのデータベース

初回は必要なファイルのダウンロードが行われるため、数分かかることがあります。

#### データベースが正しく起動したか確認する

```bash
docker compose ps
```

以下のように、2つのサービスが「running」と表示されていれば成功です。

```
NAME              STATUS
graphrag-neo4j    running
graphrag-qdrant   running
```

#### データベースを停止する場合

パソコンをシャットダウンする前や、システムを使わないときは、以下のコマンドで停止できます。

```bash
docker compose down
```

次回使うときは、再度 `docker compose up -d` で起動してください。データは保持されますので、登録したドキュメントが消えることはありません。

### ステップ 4: システムのインストール

```bash
uv sync
```

このシステムが動作するために必要な部品（ライブラリ）がすべて自動的にインストールされます。初回は、AI モデルのダウンロードも含まれるため、5分から10分程度かかることがあります。進行状況が画面に表示されますので、完了するまでお待ちください。

### ステップ 5: 動作確認

```bash
uv run streamlit run app.py
```

このコマンドを実行すると、ブラウザが自動的に開き、ドキュメント管理画面が表示されます。

もしブラウザが自動で開かない場合は、お使いのブラウザで以下のアドレスを直接入力してください。

```
http://localhost:8501
```

画面の左側（サイドバー）に「Neo4j」と「Qdrant」の接続状態が緑色で表示されていれば、セットアップは完了です。

この管理画面を終了するには、ターミナルで `Ctrl + C`（Control キーを押しながら C キー）を押してください。

## 4. ドキュメントの登録方法

### Web UI から登録する（推奨）

最も簡単な方法です。

1. ターミナルで以下のコマンドを実行して管理画面を起動します。

   ```bash
   uv run streamlit run app.py
   ```

2. ブラウザで `http://localhost:8501` が開きます。

3. **「アップロード」タブ**をクリックします。

4. PDF ファイルまたは Markdown ファイルを画面上の「ファイルをドラッグ＆ドロップ」エリアにドラッグするか、「Browse files」ボタンをクリックしてファイルを選択します。複数のファイルを一度にアップロードすることもできます。

5. **カテゴリ**を入力します。カテゴリは、ドキュメントを整理するためのラベルです。たとえば「法制度/後見」「福祉/介護」「判例/民事」のように、お好みの分類名を自由に付けられます。

6. **「取り込み開始」ボタン**をクリックします。

7. 処理が完了すると、「○○チャンクを取り込みました」というメッセージが表示されます。Gemini API キーを設定している場合は、エンティティ（人物名・組織名など）も自動的に抽出されます。

### 「一覧」タブで確認する

登録済みのドキュメントは、「一覧」タブで確認できます。各ドキュメントのタイトル、カテゴリ、チャンク数、エンティティ数が表示されます。不要なドキュメントはこの画面から削除することもできます。

### 「エンティティ」タブで関係性を見る

Gemini API を有効にしている場合、「エンティティ」タブで抽出された人物・組織・概念の一覧を確認できます。エンティティを選択すると、関連するエンティティやドキュメントが表示されます。

### Markdown ファイルの書き方

すでにお持ちの Markdown ファイルをそのまま登録できますが、以下のように先頭に「メタデータ」を書くと、システムがより正確にドキュメントを分類・整理できます。

```markdown
---
title: 成年後見制度の概要
category: 法制度/後見
---

# 成年後見制度の概要

成年後見制度は、判断能力が不十分な方の権利を守るための制度です。
家庭裁判所によって選ばれた成年後見人等が、本人の利益を考えながら、
本人を代理して契約などの法律行為を行います。

## 制度の種類

### 法定後見制度
...
```

先頭の `---` で囲まれた部分が「メタデータ」（整理情報）です。`title` と `category` を書くだけで、システムが自動的にドキュメントを分類・整理します。メタデータがなくても登録は可能です。その場合、ファイル名がタイトルとして使われます。

## 5. Claude Desktop からの利用方法（MCPサーバー連携）

Claude Desktop と連携すると、自然な日本語で会話するだけで、登録したドキュメントの検索やエンティティの探索ができるようになります。これが、プログラミングの知識がない方にとって最も便利な使い方です。

### 設定方法

#### 手順 1: Claude Desktop をインストールする

まだインストールしていない場合は、https://claude.ai/download からダウンロードしてインストールしてください。

#### 手順 2: 設定ファイルを編集する

Claude Desktop の設定ファイルを編集します。このファイルの場所はお使いの OS によって異なります。

- **Mac**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

> **Mac での設定ファイルの開き方**: Finder のメニューバーから「移動」>「フォルダへ移動」を選び、`~/Library/Application Support/Claude/` と入力してください。その中にある `claude_desktop_config.json` をテキストエディタ（テキストエディット等）で開きます。
>
> ファイルが存在しない場合は、Claude Desktop を一度起動してから再度確認してください。それでもない場合は、新しいファイルとして作成してください。

#### 手順 3: 以下の内容を設定する

設定ファイルに以下の内容を記述します。すでに内容がある場合は、`mcpServers` の中に `graphrag` の項目を追加してください。

```json
{
  "mcpServers": {
    "graphrag": {
      "command": "uv",
      "args": ["run", "--directory", "/path/to/graphrag-hybrid", "python", "server.py"],
      "env": {}
    }
  }
}
```

**重要**: `/path/to/graphrag-hybrid` の部分を、ステップ 1 でダウンロードした実際のフォルダのパスに書き換えてください。

パスの確認方法は以下のとおりです。

- **Mac / Linux**: ターミナルで `graphrag-hybrid` フォルダに移動してから `pwd` と入力すると、パスが表示されます
- **Windows**: エクスプローラーで `graphrag-hybrid` フォルダを開き、アドレスバーに表示されるパスをコピーしてください（`\` を `/` に置き換えてください）

たとえば Mac で、ホームフォルダにダウンロードした場合は以下のようになります。

```json
{
  "mcpServers": {
    "graphrag": {
      "command": "uv",
      "args": ["run", "--directory", "/Users/tanaka/graphrag-hybrid", "python", "server.py"],
      "env": {}
    }
  }
}
```

#### 手順 4: Claude Desktop を再起動する

設定ファイルを保存したら、Claude Desktop を一度完全に終了し、再度起動してください。

起動後、Claude との会話画面の下部にハンマーのアイコンが表示されていれば、GraphRAG との連携が成功しています。アイコンをクリックすると、利用可能なツール（search、get_document、search_entities など）が一覧で表示されます。

### 使い方の例

Claude Desktop で以下のように話しかけるだけで、GraphRAG のドキュメント検索が自動的に実行されます。

#### 例 1: ドキュメント検索

> **あなた**: 「成年後見制度について、利用者の家族向けに説明できる資料を探してください」
>
> **Claude**: GraphRAG の検索ツールを使って関連するドキュメントを見つけ、内容を要約して回答します。たとえば「以下の3つの資料が見つかりました。1つ目は成年後見制度の概要で、制度の種類と申立て手続きについて説明しています...」のように、分かりやすく整理して教えてくれます。

#### 例 2: エンティティの探索

> **あなた**: 「家庭裁判所に関連する情報をすべて見せてください」
>
> **Claude**: エンティティ検索を使って「家庭裁判所」が登場するすべてのドキュメントを探し、関連するエンティティ（成年後見人、申立人、後見監督人など）との関係も表示します。

#### 例 3: 新しいドキュメントの追加

> **あなた**: 「デスクトップにある『新しい通知.md』をシステムに登録してください」
>
> **Claude**: ファイルのパスを確認した上で、ドキュメント取り込みツールを使ってシステムに登録します。登録が完了すると、チャンク数などの情報を報告してくれます。

#### 例 4: 複合的な調査

> **あなた**: 「障害者総合支援法と介護保険法の適用関係について、登録されている資料から整理してください」
>
> **Claude**: 複数の検索を組み合わせて関連資料を収集し、両制度の適用関係を整理した回答を生成します。

### 使い方のコツ

- **具体的に質問する**: 「後見制度について」よりも「成年後見制度の申立て手続きの流れについて」のように具体的に聞くと、より的確な結果が得られます
- **自然な言葉で大丈夫です**: 法律用語や正式名称を使わなくても、日常の言葉で質問できます
- **追加の質問もできます**: 最初の検索結果を踏まえて「この中で、家族が最初にすべきことは何ですか？」のように続けて質問できます

## 6. よくある質問（FAQ）

### Q: プログラミングの知識は必要ですか？

A: セットアップ時にいくつかのコマンドをターミナルで実行する必要がありますが、このガイドに記載されたコマンドをそのままコピー＆ペーストすれば動作します。セットアップ完了後は、Web UI や Claude Desktop を使って操作しますので、プログラミングの知識は不要です。

### Q: インターネット接続は必要ですか？

A: 初回セットアップ時にはインターネット接続が必要です（ソフトウェアのダウンロードとAIモデルの取得のため）。セットアップが完了した後は、基本的な検索機能はオフラインでも利用できます。ただし、Gemini API を使ったエンティティ自動抽出の機能のみ、インターネット接続が必要です。

### Q: データはどこに保存されますか？

A: すべてのデータはあなたのパソコンの中に保存されます。外部のクラウドサービスにデータが送信されることはありません。ただし、Gemini API を使ったエンティティ抽出機能を有効にしている場合は、抽出処理のためにドキュメントの内容が Google のサーバーに送信されます。機密性の高いドキュメントを扱う場合は、Gemini API を無効にした状態（`.env` ファイルで `GEMINI_API_KEY` を設定しない）でご利用ください。

### Q: どんなファイルが使えますか？

A: PDF ファイルと Markdown ファイル（拡張子 `.md`）に対応しています。Word 文書（`.docx`）は直接取り込めませんので、事前に PDF に変換してからアップロードしてください。Word の「名前を付けて保存」や「エクスポート」機能で PDF に変換できます。

### Q: 登録できるドキュメントの数に制限はありますか？

A: ソフトウェアとしての制限はありませんが、パソコンのストレージ容量とメモリに依存します。一般的な業務用パソコンであれば、数百件のドキュメントは問題なく扱えます。

### Q: 検索がうまくいかない場合は？

A: まず、データベースが動作しているかを確認してください。ターミナルで `docker compose ps` を実行し、2つのサービスが「running」になっていることを確認します。それでも解決しない場合は、次のセクション「トラブルシューティング」を参照してください。

### Q: 複数人で共有して使えますか？

A: 現在のバージョンは、1台のパソコンで個人利用することを想定しています。複数人での共有利用をご希望の場合は、サーバーへのデプロイが必要になりますので、IT に詳しい方にご相談ください。

## 7. トラブルシューティング

問題が発生した場合は、以下の一覧から症状に近いものを探してください。

### 「docker compose up」で失敗する

**考えられる原因**: Docker Desktop が起動していません。

**解決方法**: Docker Desktop アプリを起動してください。Mac の場合はアプリケーションフォルダにある Docker アイコンをダブルクリックします。画面の左下に「Engine running」と緑色で表示されるまで待ってから、もう一度 `docker compose up -d` を実行してください。

### 「uv sync」で失敗する

**考えられる原因**: uv がインストールされていないか、パスが通っていません。

**解決方法**: ターミナルで以下のコマンドを実行して、uv がインストールされているか確認してください。

```bash
uv --version
```

「command not found」と表示される場合は、uv のインストールが完了していません。「2. 必要なもの」のセクションに戻り、uv のインストールを行ってください。インストール後、ターミナルを一度閉じて開き直す必要がある場合があります。

### Streamlit の画面がブラウザで開かない

**考えられる原因**: ブラウザが自動で起動しなかった、またはポートが他のアプリケーションに使用されています。

**解決方法**: お使いのブラウザのアドレスバーに `http://localhost:8501` と手動で入力してください。それでも表示されない場合は、ターミナルに表示されたメッセージを確認してください。別のポート番号が表示されている場合があります。

### サイドバーに「Neo4j: 未接続」と表示される

**考えられる原因**: データベースがまだ起動完了していないか、停止しています。

**解決方法**: ターミナルで `docker compose ps` を実行してください。`graphrag-neo4j` のステータスが「running」であることを確認します。起動直後は準備に30秒ほどかかることがありますので、少し待ってから管理画面の「再接続」ボタンをクリックしてください。

### 検索結果が 0 件になる

**考えられる原因**: ドキュメントがまだ登録されていません。

**解決方法**: 管理画面の「一覧」タブを開き、ドキュメントが登録されているか確認してください。ドキュメントが表示されない場合は、「アップロード」タブからドキュメントを登録してください。

### Claude Desktop でハンマーのアイコンが表示されない

**考えられる原因**: 設定ファイルの記述に誤りがあるか、パスが間違っています。

**解決方法**: 以下を確認してください。

1. 設定ファイル（`claude_desktop_config.json`）が正しい場所にあること
2. JSON の記述が正しいこと（カンマの過不足、括弧の閉じ忘れなどに注意）
3. `/path/to/graphrag-hybrid` の部分が、実際のフォルダのパスに置き換えられていること
4. 設定を保存した後、Claude Desktop を完全に終了して再起動したこと
5. Docker でデータベースが起動していること（`docker compose up -d`）

### エンティティが抽出されない

**考えられる原因**: Gemini API キーが設定されていません。

**解決方法**: これは正常な動作です。エンティティ抽出機能を使うには、`.env` ファイルに Gemini API キーを設定する必要があります。この機能は任意ですので、設定しなくてもセマンティック検索とグラフ検索は正常に動作します。

## 8. 用語集

このガイドや管理画面に登場する技術用語を、分かりやすく説明します。

### MCP（Model Context Protocol）

AI アシスタント（Claude など）が外部のツールやデータにアクセスするための通信規格です。このシステムでは、Claude Desktop から GraphRAG の検索機能を呼び出すために使われています。電話の「通話規格」のようなものだとお考えください。Claude とシステムが「会話」するためのルールです。

### セマンティック検索

単語の完全一致ではなく、文章の「意味」を理解して検索する方式です。たとえば「車」と「自動車」が同じ意味であることを理解します。従来のキーワード検索と異なり、言い回しが違っても意味が近い文書を見つけることができます。

### ナレッジグラフ

情報同士の関係性をネットワーク状に表現したデータ構造です。たとえば「成年後見制度」は「家庭裁判所」で「申立てる」、「成年後見人」が「被後見人」を「代理する」といった関係を蜘蛛の巣のようにつなげて管理します。

### エンティティ

テキストから抽出された重要な固有名詞や概念のことです。人名、組織名、法律名、制度名、技術名などが該当します。システムがドキュメントの内容を自動的に読み取り、これらを識別・分類します。

### チャンク

長いドキュメントを検索しやすいサイズに分割した断片のことです。たとえば50ページある報告書を、段落や見出し単位で区切った小さなブロックにします。これにより、検索時に「報告書全体」ではなく「関連する部分」をピンポイントで見つけることができます。

### ベクトル

テキストの意味を数値の列（たとえば 768 個の数値の並び）として表現したものです。AI が文章の意味を理解するための内部的な形式で、意味が似ている文章は似た数値の列になります。利用者が意識する必要はありませんが、セマンティック検索の仕組みを支える基盤技術です。

### Docker

アプリケーションの実行環境をパッケージ化するツールです。このシステムでは、Neo4j と Qdrant という2つのデータベースを、Docker を使って簡単に起動・停止できるようにしています。「データベース専用の小さなパソコンをあなたのパソコンの中に作る」ようなイメージです。

### Neo4j

グラフデータベースと呼ばれる種類のデータベースです。情報同士の「つながり」を効率的に管理することに特化しています。このシステムでは、ドキュメント同士の関係性やエンティティのつながりを管理するために使われています。

### Qdrant

ベクトルデータベースと呼ばれる種類のデータベースです。テキストの「意味」を数値化したデータを効率的に検索することに特化しています。このシステムでは、セマンティック検索を実現するために使われています。

### Streamlit

Python で Web アプリケーションを簡単に作成するためのフレームワークです。このシステムのドキュメント管理画面（Web UI）は Streamlit で作られています。利用者が Streamlit について詳しく知る必要はありません。
